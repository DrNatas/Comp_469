1. Rule compression: propositional logic vs first-order logic

Propositional logic needs a separate breeze rule for each square:

B_1_1 ⇔ (P_1_2 ∨ P_2_1)
B_1_2 ⇔ (P_1_1 ∨ P_1_3 ∨ P_2_2)
B_1_3 ⇔ (P_1_2 ∨ P_1_4 ∨ P_2_3)
...

First-order logic says it once with variables and quantifiers:

∀s Breezy(s) ⇔ ∃p (Pit(p) ∧ Adjacent(p,s))

Class prompt: How many rules for a 100 x 100 grid? Propositional: 10,000. FOL: one general schema.

2. Quantifier Quest

Change the world and compare two sentences:

∀x ∃y Loves(x,y) means everybody loves somebody.

∃y ∀x Loves(x,y) means there is one person loved by everybody.

5. Database semantics checkpoint

Known facts: Brother(John, Richard), Brother(Geoffrey, Richard), and John ≠ Geoffrey.

Standard FOL semantics

Can Richard have another brother?

UNKNOWN

The KB does not rule out extra objects or extra facts.

Database semantics

Are John and Geoffrey the only brothers?

TRUE

Unique names + closed world + domain closure make the answer definite.
